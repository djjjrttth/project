# Идея и презентация проекта

## 1. Цель проекта

Разработать 2D-игру **Tomb of the Mask - Student Edition** для персонального компьютера на языке C++ с использованием библиотеки SFML.

Игрок проходит лабиринты, собирает монеты, использует бонусы времени и должен добраться до выхода до окончания таймера.

---

## 2. Актуальность

* Популярность аркадных игр и лабиринтов;
* Практическое применение объектно-ориентированного программирования;
* Изучение работы с графикой и игровыми циклами в SFML;
* Освоение командной разработки и системы контроля версий Git.

---

## 3. Что делает программа

Игра представляет собой набор лабиринтов с ограничением по времени.

Игрок перемещается по клеткам, собирает монеты, получает бонусы времени и стремится достичь выхода до окончания отсчёта.

Доступны два режима:

* **Level Mode** — прохождение заранее подготовленных уровней;
* **Endless Mode** — бесконечный режим со случайной генерацией лабиринтов.

# Реализация и работа программы

## Основные функции игры «Tomb of the Mask - Student Edition»

---

### 1. Игровое меню

Главное меню содержит три пункта:

* **Level Mode**
* **Endless Mode**
* **Exit**

Управление осуществляется клавишами:

* **W/S** или стрелками;
* **Enter** — подтверждение выбора;
* **1, 2, 3** — быстрый выбор пункта меню.

Для оформления используется отдельный фон в стиле Backrooms.

---

### 2. Система уровней

Режим **Level Mode** использует заранее подготовленные карты:

* `levels/level1.txt`
* `levels/level2.txt`
* `levels/level3.txt`

Каждый уровень содержит:

* стены;
* свободные клетки;
* стартовую позицию игрока;
* монеты;
* бонусы времени;
* выход.

После прохождения уровня выполняется переход к следующему.

---

### 3. Бесконечный режим

Режим **Endless Mode** создаёт новые лабиринты автоматически.

Особенности:

* процедурная генерация;
* использование алгоритма Прима;
* бесконечное прохождение;
* сохранение лучшего результата текущей игровой сессии.

---

### 4. Игрок (Player)

Класс Player отвечает за:

* позицию персонажа;
* обработку перемещения;
* отрисовку спрайта.

Особенности реализации:

* управление клавишами **WASD** или стрелками;
* перемещение на **одну клетку за одно нажатие клавиши**;
* проверка столкновений со стенами;
* использование текстуры `player.png`.

---

### 5. Игровые объекты

На карте размещаются различные объекты.

#### Монеты

* обозначаются символом `.`;
* увеличивают счёт игрока;
* исчезают после подбора.

#### Бонусы времени

* обозначаются символом `B`;
* добавляют время к таймеру;
* исчезают после использования.

#### Выход

* обозначается символом `E`;
* завершает текущий уровень.

---

### 6. Игровой таймер

На каждом уровне действует ограничение по времени.

Функциональность:

* отсчёт времени во время игры;
* отображение оставшегося времени;
* завершение игры при окончании таймера;
* возможность увеличения времени за счёт бонусов.

Для реализации используется отдельный класс **GameTimer**.

---

### 7. Пользовательский интерфейс (UI)

На экране отображаются:

* текущий счёт;
* оставшееся время;
* лучший результат в Endless Mode;
* игровые сообщения.

Дополнительно реализованы:

* экран победы;
* экран поражения;
* элементы меню;
* анимация перехода между уровнями.

---

### 8. Анимация перехода между уровнями

После завершения уровня отображается специальная анимация открытия двери.

Назначение:

* визуально разделять уровни;
* улучшать восприятие игрового процесса.

---

### 9. Основной игровой цикл

Программа построена по классической схеме игрового цикла:

```text
pollEvent -> update -> render
```

Этапы работы:

1. Обработка пользовательского ввода;
2. Обновление состояния игры;
3. Проверка условий победы и поражения;
4. Отрисовка всех объектов.

---

### 10. Обработка событий

Поддерживаются следующие действия:

#### Во время игры

* **WASD** или стрелки — движение;
* **R** — перезапуск текущего уровня;
* **Esc** — возврат в главное меню;
* **Enter** — переход к следующему уровню после победы.

#### В меню

* **W/S** или стрелки — выбор пункта;
* **Enter** — подтверждение;
* **1/2/3** — быстрый выбор режима.

---

### 11. Формат файлов уровней

Для описания уровней используются текстовые файлы.

| Символ | Значение      |
| ------ | ------------- |
| #      | Стена         |
| Пробел | Пустая клетка |
| P      | Старт игрока  |
| E      | Выход         |
| .      | Монета        |
| B      | Бонус времени |

Шипы были исключены из финальной версии проекта для упрощения игрового процесса и повышения предсказуемости поведения уровней.

## Сборка проекта

### CMake

```bash
cmake -S . -B build -DSFML_ROOT=C:/SFML-2.6.1
cmake --build build --config Release
```

Запуск:

```bash
./build/TombOfTheMaskStudentEdition.exe
```

Для MinGW также может использоваться:

```bash
build_windows_mingw.bat
```

---

## Возможные проблемы при запуске

Если появляется ошибка `0xc000007b`, необходимо:

1. Удалить старую папку `build`;
2. Не использовать DLL-файлы от других версий SFML;
3. Не смешивать 32-битные и 64-битные библиотеки;
4. Не смешивать Debug и Release версии DLL;
5. Использовать сборку SFML для того же компилятора, которым собирается проект.
